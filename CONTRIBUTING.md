# Contributing

Happy to see that you might be interested in contributing to this project.



## Open an issue

If you have an idea, suggestions, or found a bug, please open an issue and try to be as descriptive as possible.

## Developing and contributing code

### Set up locally

Set up locally as described in the README.

### Code quality

Make sure your changes pass the pre-commit hooks. You can also run them without commiting with: pre-commit run --all-files

### Branching and commits

Please use a descriptive name for your branch and create a PR to the projects develop branch. Please also give your commits good descriptions.

## Testing
If you make changes to the infrastructure/application. Please verify that the automated emails still work as expected.
